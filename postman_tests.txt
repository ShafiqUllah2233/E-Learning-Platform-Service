LEARN-SPHERE API TESTS
=====================

1. COURSE SERVICE (Port 3001)
----------------------------

a) Create Course
POST http://localhost:3001/api/courses
Headers:
- Content-Type: application/json
- Authorization: Bearer YOUR_JWT_TOKEN
Body:
{
    "title": "Introduction to Node.js",
    "description": "Learn Node.js fundamentals",
    "instructor": "John Doe",
    "category": "Programming",
    "price": 49.99
}

b) Get All Courses
GET http://localhost:3001/api/courses

c) Get Course by ID
GET http://localhost:3001/api/courses/:courseId


2. STUDENT SERVICE (Port 3002)
-----------------------------

a) Register Student
POST http://localhost:3002/api/students/register
Headers:
- Content-Type: application/json
Body:
{
    "name": "Alice Smith",
    "email": "alice@example.com",
    "password": "password123",
    "preferences": ["Programming"]
}

b) Login Student
POST http://localhost:3002/api/students/login
Body:
{
    "email": "alice@example.com",
    "password": "password123"
}

c) Get Student Profile
GET http://localhost:3002/api/students/profile
Headers:
- Authorization: Bearer YOUR_JWT_TOKEN


3. ENROLLMENT SERVICE (Port 3003)
--------------------------------

a) Create Enrollment
POST http://localhost:3003/api/enrollments
Headers:
- Content-Type: application/json
- Authorization: Bearer YOUR_JWT_TOKEN
Body:
{
    "studentId": "STUDENT_ID_FROM_REGISTRATION",
    "courseId": "COURSE_ID_FROM_CREATION"
}

b) Get Student Enrollments
GET http://localhost:3003/api/enrollments/student/:studentId
Headers:
- Authorization: Bearer YOUR_JWT_TOKEN

c) Get Enrollment by ID
GET http://localhost:3003/api/enrollments/:enrollmentId
Headers:
- Authorization: Bearer YOUR_JWT_TOKEN


TESTING FLOW
============

1. Start all services:
   Open three terminals and run:
   - cd course-service && npm start
   - cd student-service && npm start
   - cd enrollment-service && npm start

2. Test Database Connection:
   - Open MongoDB Compass
   - Connect to: mongodb://localhost:27017
   - Check for databases:
     * learn-sphere-courses
     * learn-sphere-students
     * learn-sphere-enrollments

3. Testing Sequence:
   a) Register a student (save the JWT token)
   b) Create a course (using JWT token)
   c) Create enrollment using student and course IDs
   d) Verify enrollment in database

ERROR CHECKING
=============
- Check response status codes (200, 201, 400, 401, 404, 500)
- Verify data in MongoDB Compass after each operation
- Check console logs for each service

# LEARN-SPHERE API TESTING SEQUENCE

## 1. STUDENT SERVICE (Port 3002)
-----------------------------------

### 1.1 Register Student
POST http://localhost:3002/api/students/register
Headers:
  Content-Type: application/json
Body:
{
    "name": "Test Student",
    "email": "student@test.com",
    "password": "password123",
    "preferences": ["Programming", "Web Development"]
}
Expected Response: 201 Created with token

### 1.2 Login Student
POST http://localhost:3002/api/students/login
Headers:
  Content-Type: application/json
Body:
{
    "email": "student@test.com",
    "password": "password123"
}
Expected Response: 200 OK with token
⚠️ Save the token for subsequent requests!

### 1.3 Get Student Profile
GET http://localhost:3002/api/students/profile
Headers:
  Authorization: Bearer <token_from_login>
Expected Response: 200 OK with student details

## Testing Tips
-----------------------------------
1. Start all services before testing:
   - MongoDB service
   - Course service (Port 3001)
   - Student service (Port 3002)
   - Enrollment service (Port 3003)

2. Test sequence matters:
   - Register → Login → Create Course → Create Enrollment
   - Save IDs and tokens from responses
   - Use saved values in subsequent requests

3. Common Issues:
   - "Unauthorized": Check token format (Bearer prefix)
   - "Not Found": Verify service is running and IDs are correct
   - "MongoDB Error": Check database connection

4. Expected Status Codes:
   - 201: Resource Created
   - 200: Success
   - 400: Bad Request
   - 401: Unauthorized
   - 404: Not Found
   - 500: Server Error
